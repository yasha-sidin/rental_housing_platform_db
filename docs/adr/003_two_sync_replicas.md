# ADR 003: Две synchronous replicas

## Решение

Настроить запись так, чтобы транзакция подтверждалась только при наличии primary и минимум двух synchronous replicas.

## Обоснование

Так проект демонстрирует максимальную защиту подтвержденных транзакций при отказе primary.

## Последствия

- Если synchronous replicas меньше двух, запись останавливается.
- Это повышает надежность, но уменьшает доступность записи при деградации кластера.
