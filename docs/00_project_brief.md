# Project Brief

Проект для курса OTUS "Базы данных": построение отказоустойчивого PostgreSQL-кластера для платформы краткосрочной аренды жилья.

## Цель

Показать связанную систему:

- доменную модель базы данных;
- понятную цепочку миграций;
- отказоустойчивый PostgreSQL-кластер;
- `RPO = 0` для подтвержденных транзакций при отказе primary;
- backup/PITR для защиты от логической порчи данных;
- демонстрационные сценарии для защиты.

## Границы

Проект не реализует полноценное приложение аренды жилья. Клиентский слой представлен демонстрационными подключениями через PgBouncer/HAProxy и SQL-сценариями.

Docker-стенд демонстрирует архитектурные принципы, но production-требования описываются отдельно: failure domains, надежное S3-compatible хранилище, secrets, мониторинг, alerting и runbooks.

## Ключевые решения

- PostgreSQL HA строится на Patroni.
- DCS/quorum контур строится на 5 etcd-узлах.
- Запись требует primary и минимум 2 synchronous replicas.
- Клиенты используют локальные PgBouncer + HAProxy, чтобы proxy одного клиента не становился SPOF для остальных.
- Миграции применяются через writer endpoint после поднятия HA-кластера.
- Backup делает отдельный backup-контур из двух workers.
- Для демонстрации используется full backup + WAL archive + PITR.
